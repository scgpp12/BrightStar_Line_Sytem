# 営業アシスタント（要員通勤コスト比較）利用マニュアル

**BrightStar 統合 LINE アシスタント — 営業チャネル**
対象：営業部 担当者 / 版本：v1.1 / 最終更新：2026-06-19

> 本チャネルは **営業部専用** です。現場（駅）に対する待機要員の通勤コストを即座に比較し、要員名簿（CSV / 単票）の保守も LINE 上で行えます。
> 中文：本窗口仅限**营业部**使用；可比较待机要员到现场的通勤成本，并在 LINE 上维护要员名册。

---

## 1. はじめに（毎日の本人確認）

### 1-1. 初回登録（最初の1回だけ）
```
営業部 田中
```
- 社員名簿と照合し、**営業部**の本人のみ登録できます（同姓同名は `営業部 田中 E002` のように社員番号付与）。
- LINE アカウントと社員番号は**専用に紐付き**ます。

### 1-2. 毎日の認証 / 言語
2日目以降は **「認証」** ワンタップ。認証後その日の最初に **「日本語」／「中文」**（既定：日本語）を選択。

### 1-3. 登録解除
**「登録解除」** で紐付け解除 → 別人で再登録。

## 2. 通勤コスト比較（メイン機能）

### 2-1. 検索のしかた
誤作動防止のため、**明示プレフィックス**を付けたときだけ比較を実行します。

```
検索路線：東京
检索路线：东京
検索路線 新宿        ← 全角／半角コロン・スペース区切りも可
route: Shinjuku
```

- 対応プレフィックス：`検索路線` / `路線検索` / `通勤比較` / `現場` / `検索路线` / `路线检索` / `通勤比较` / `现场` / `路线` / `route` / `commute` など。
- 駅名だけ（例：`東京`）や、それ以外の自由入力では比較は走らず、**操作の案内**が返ります。
- 駅名は **日本語・かな・ローマ字**に対応（中国語表記はそのままでは未対応）。

### 2-2. 返信の見かた
```
■ 現場「赤羽」への通勤コスト（待機中4名 / 定期代が安い順）

・拉拉（池袋→赤羽）: 8分/乗換0回 IC199円 定期5,890円
　経由: 池袋→[JR埼京線]→赤羽
　🔎確認: https://ekitan.com/transit/fare/sf-.../st-...
...
```
- **定期代（1ヶ月）が安い順**に並びます。
- 各行に **「🔎確認」リンク**（ekitan）が付き、ブラウザで経路・運賃を自分で再確認できます。
- 比較のあと、**「結果CSVダウンロード」ボタン**が届きます。Excel で開いて保存・共有できます。

### 2-3. 「経由」の読み方 — 乗換回数と駅数が違う理由
経由は **「駅 →[路線名]→ 駅」** で表示し、駅間を歩いて乗り換える区間は **〔徒歩〕** で示します。

```
経由: 西大島→[都営新宿線]→神保町→[都営三田線]→春日〔徒歩〕後楽園→[南北線]→赤羽岩淵〔徒歩〕赤羽
```
- 上の例は **乗換2回**ですが、経由には駅名が多く並びます。
- これは **〔徒歩〕＝駅間の徒歩連絡**（例：春日↔後楽園、赤羽岩淵↔赤羽）が含まれるためです。
- **徒歩連絡は「乗換回数」には数えません**（電車の乗換だけを数えます）。だから経由の駅数より乗換回数が少なくなります。

> 補足：同じ現場・同じ最寄駅の組み合わせは結果をキャッシュします。初回だけ取得に少し時間がかかり、結果は後追いで届きます（push）。

## 3. 要員名簿の保守

### 3-1. CSV で一括編集（ダウンロード → 編集 → 送り返す）
1. **「要員一覧DL」** と送ると、CSV のダウンロードボタンが届きます。
2. ダウンロードし、**Excel で編集**します（追加・変更）。
3. 編集した CSV ファイルを LINE に**送り返す**と反映されます。

**CSV の列**
```
staff_id, name, nearest_station, address, department, status, updated_at
```
- `nearest_station` … 最寄駅。`address`（住所）を書くと**アップロード時に最寄駅へ自動変換**します（住所自体は保存しません）。
- `updated_at` … 楽観ロック用の管理列。**編集しないでください**。

**住所にカンマが含まれる場合の注意**
- 住所は `"東京都江東区大島3-4-7, ..."` のように **半角ダブルクォート `"` で囲って**ください（Excel で編集すれば自動で付きます）。
- スマホの**弯曲した引用符 `" "` は無効**です。半角 `"` を使ってください。
- ダウンロードした CSV は全項目が `"` で囲まれています（カンマを含む住所も安全）。
- カンマで列がずれた行は**自動検知して取り込みを止め**、該当行を知らせます。

### 3-2. 競合防止（楽観ロック）
- アップロード時、**自分がダウンロードした後に他の人が更新していないか**を `updated_at` で確認します。
- 競合があれば取り込みを**拒否**し、「最新を再ダウンロードしてから編集してください」と案内します。
- → 複数人が同時に編集しても、上書きで消える事故を防ぎます。

### 3-3. 単票での増減（カンマ住所も安全な `|` 区切り）
1 件だけ素早く直すときは **`|`（縦棒）区切り**のコマンドが便利です（住所のカンマを気にせず入力できます）。

| 操作 | 送信例 |
|------|--------|
| 追加 | `追加\|E007\|新人太郎\|大宮\|営業部\|available` |
| 追加（住所→駅変換） | `追加\|E007\|新人太郎\|東京都江東区大島3-4-7, ...\|営業部\|available` |
| 変更 | `変更\|E003\|品川\|assigned` |
| 削除 | `削除\|E003` |
| 照会 | `照会\|E003` |

- `最寄駅` 欄に**カンマを含む住所**を入れると、自動で最寄駅に変換されます。
- `status` の値：`available`（待機中）/ `assigned`（稼働中）など。待機中の要員だけが通勤比較の対象になります。

## 4. よくある質問

**Q. 「照会」と送っても何も起きません。**
比較は `検索路線：東京` のようにプレフィックスが必要です。1件照会は `照会|E003` のように `|` を付けてください。

**Q. CSV をアップしたら列がずれていると言われました。**
住所のカンマが原因です。住所を半角 `"` で囲むか、`追加|…` / `変更|…` の単票コマンドをご利用ください。

**Q. アップロードが「競合」で拒否されました。**
他の人が先に更新しています。「要員一覧DL」で最新を取り直してから、編集・再アップしてください。

**Q. 中国語の駅名で検索できません。**
駅名は日本語・かな・ローマ字で送ってください。
