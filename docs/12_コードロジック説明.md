## 2. 共通モジュール

### 2-1. `authlib.py` — 認証ゲート
- `gate(channel, uid, text, role_pred) -> (action, item)`：
  本人確認の中核。花名册照合 → 役割述語(`role_pred`)で判定し、`pass/ok/tap/need_bind/not_found/ambiguous/wrong_role/taken` を返す。
- `bind_line(empId, uid)`：他の紐付けを解除してから排他登録（占用ロック）。
- `unbind(uid)`：花名册の `lineUserId` と認証行を全消去。
- `grant_temp(channel, uid, seconds)`：`validUntil` を書く（テスト用バックドア）。
- `is_authed`：`validUntil` または `authedDate==today(JST)` で当日有効を判定。
- `find_by_line(uid)` / `master_code_today(prefix)` / `today_jst()`。

### 2-2. `assist.py` — 多言語・言語設定・ヘルプ
- `resolve(text, intents)`：テキスト完全一致（大小無視）で canonical intent に解決。多言語同義語を 1 つに畳む。
- `get_lang/set_lang/needs_lang_today(channel, uid)`：認証(auth)テーブルの `lang`/`langDate` を読み書き。
- `detect_lang_word(text)`：「日本語／中文」等の言語選択語を検出。
- `quick_reply(text, items)` / `lang_chooser(name)` / `help_message(lang, title, entries)`：Quick Reply メッセージを組み立て。
- `COMMON_INTENTS`：help / auth / reset を共有。

### 2-3. 共有の配置方法
- `assist.py` / `authlib.py` は各チャネルのビルド時にコピー（eigyo は `infra/build.py` が `build/lambda` へ複製）。
- DynamoDB のテーブル名は **環境変数**（`AUTH_TABLE` / `ROSTER_TABLE` 等）で受け渡し、コードに固定しない。

## 3. チャネル別フロー

### 3-1. 共通の受信ループ
```
署名検証(verify_signature) → events をループ
  → 登録解除(RESET_WORDS) ?
  → gate() で本人確認（action != ok/pass なら案内を返す）
  → 言語選択語 → set_lang → 認証OK+ヘルプ
  → needs_lang_today → 言語チューザー
  → resolve(text, INTENTS) で機能分岐
```
- 返信は **reply_token**（一度きり・約30秒）。Quick Reply 等は `reply_messages`。

### 3-2. 研修（kenshu）`src/handlers/line_webhook.py`
- LINE 前面は **講師(teacher)専用ゲート**（`_teacher_pred`）。受講者は社員アシスタントへ誘導。
- 認証後 `webhook._route(ev)` に委譲し、建课/发布/改课/删课/学员列表/名单/分组 を処理。
- `_handle_event` は str（テキスト）または dict（Quick Reply）を返し、ハンドラが種別で振り分ける。

### 3-3. 人事（jinji）`lambda/handlers/line_webhook.py`
- `_route`：バックドア → reset → gate(HR) → 言語フロー → `resolve(INTENTS)`。
- INTENTS：roster_status / missing / remind / bulk_dl / roster_admin / proofread（多言語）。
- `business.roster_of(uid)` は **roster.lineUserId** を優先（identity の単一情報源）。
- 催促は別 Lambda（`REMINDER_FUNCTION_NAME`）を非同期 invoke。提出済は差集合で除外。

### 3-4. 営業（eigyo）`infra/lambda_app/line_handler.py`
処理順（認証後）：
1. 言語選択 → 認証OK+ヘルプ
2. `needs_lang_today` → 言語チューザー
3. `help`
4. CSV ファイル受信 → `_import_staff_csv`（**列ずれ検知 → 楽観ロック → 住所→駅変換 → bulk_upsert**）
5. `要員一覧DL` → `_export_to_s3`（**QUOTE_ALL** で全項目引用）+ 署名付き DL リンク
6. `|` を含む → 単票 CRUD（`_handle_crud`）
7. **明示プレフィックス**（`_extract_site`）のときだけ現場比較。無ければ案内のみ
8. 比較は ack(reply) → 自己 invoke の `_worker` で重い取得を実行し **push**（結果＋CSVダウンロードボタン）

通勤比較の中核：
- `batch.compare_site(site, staffs, source, registry)`：各要員の最寄駅→現場を取得。
  同一 (from_id, to_id) は実行内メモでキャッシュ。`StaffCommute` に `from_id/to_id` を保持し確認URLを生成。
- `ekitan_source.EkitanScraper`：運賃ページ＋定期ページを取得し路線シグネチャで突合。
  `_route_summary` は「駅→[路線名]→駅」を組み立て、徒歩連絡を〔徒歩〕で明示（乗換回数と駅数の差を説明可能に）。
- `transit/geo.py nearest_station(住所)`：Amazon Location geo-places で住所→最寄駅。
- `DynamoDBCache`：経路結果をキャッシュ（cacheKey = `namespace#from_to`、TTL あり）。

### 3-5. 社員（shain）`lambda/handler.py`
- `from jinji.common import assist/authlib/business`、`from kenshu... import ...`（**名前空間分離 vendoring**）。
- フロー：reset → 初回登録ゲート(`jbiz.handle_registration`) → 言語フロー → ヘルプ → モード切替 → ファイルは人事固定 → 当日モードで研修/人事へ委譲。
- 研修委譲時は `_ensure_kenshu_student(uid, name)` で受講者を自動プロビジョニング（二重登録防止）。
- セッション（mode + 日付）は専用テーブル（`SHAIN_SESSION_TABLE`）に保持。

## 4. vendoring（名前空間分離）の要点

- kenshu と jinji は `common` / `handlers` のモジュール名が衝突する。社員 Lambda では
  両者を `kenshu.*` / `jinji.*` のサブパッケージに収め、**絶対 import を sed で書換え**
  （`from common` → `from kenshu.common`）、相対 import は据え置く。
- これにより 2 つの Python バックエンドを 1 Lambda に同梱しても衝突しない。

## 5. データモデル（DynamoDB）

| テーブル | キー | 主な属性 |
|----------|------|----------|
| 花名册 roster | `empId` (PK) | name / department / role / lineUserId |
| 認証 auth | `pk = channel#userId` | authedDate / lang / langDate / validUntil / expireAt(TTL) |
| courses（研修） | courseId | title / startTime / durationMin / capacity / status / zoom* / remind1hSent |
| enrollments（研修） | (openid, courseId) | status / enrolledAt |
| submissions（人事） | userId + sk=period#type | s3Key / fileName / submittedAt |
| StaffTable（営業） | staff_id | name / nearest_station / department / status / updated_at |
| CacheTable（営業） | cacheKey | payload(JSON) / cachedAt / ttl |

- 営業の `updated_at` は **マイクロ秒精度**（同一秒内更新でも楽観ロックが機能するため）。

## 6. テスト

- `tests/test_authlib.py`：gate の各状態、一括認証、占用ロック、unbind の排他、temp grant / master code を pytest + moto で検証。
