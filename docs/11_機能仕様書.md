# 機能仕様書 — BrightStar 統合 LINE アシスタント

版本：v1.0 / 最終更新：2026-06-19

本書は 4 チャネル共通機能と、各チャネル固有機能の仕様をまとめる。

---

## 1. 共通機能

### 1-1. 日次本人確認（認証ゲート）
- 入力「所属部署 お名前」を花名册(roster)と照合し、`role` を判定。
- 役割（role）：`employee` / `hr` / `teacher` / `sales`。チャネルごとに通す役割を限定。
- ゲートの判定結果（action）：

| action | 意味 | 応答 |
|--------|------|------|
| `ok` / `pass` | 認証成功・通過 | 機能利用へ |
| `need_bind` | 未紐付け | 「所属部署 お名前」を要求 |
| `not_found` | 名簿に該当なし | 人事へ連絡を案内 |
| `ambiguous` | 同部署同姓名が複数 | 社員番号の付与を要求 |
| `wrong_role` | 役割が不一致 | 該当チャネルの案内 |
| `tap` | 紐付け済み・当日未認証 | 「認証」タップを促す |
| `taken` | 番号が別アカウントで使用済み | 人事へ連絡を案内 |

- **占用ロック**：`bind_line(empId, userId)` は他の紐付けを解除してから排他的に登録。
- **登録解除**：`unbind(userId)` で花名册の `lineUserId` と認証行をすべてクリア。
- **テスト用バックドア**（人事のみ・任意）：`sonsik<YYYYMMDD>` で 1 時間の臨時 HR 権限（`MASTER_HR_PREFIX` 設定時のみ有効）。

### 1-2. 言語設定
- 認証後その日の最初に「日本語／中文」を選択（既定：日本語）。`auth` テーブルに `lang` / `langDate` を保持。
- キーワードは多言語（日・英・韓・中）の同義語を `resolve()` で canonical intent に解決。

### 1-3. ヘルプ（Quick Reply）
- 機能名とキーワードをボタン化。タップすると当該キーワードが自動送信される。

## 2. 研修（kenshu）— 講師向け

| 機能 | 入力 | 出力 |
|------|------|------|
| コース作成 | `建课 タイトル\|日時\|所要分\|定員\|紹介` | 下書き(draft)作成 |
| コース公開 | `发布 コース名` | Zoom 会議作成 + 公開、主催/受講リンク |
| コース変更 | `改课 コース名\|項目=値;…` | 該当項目を更新 |
| コース削除 | `删课 コース名` | 削除 |
| 受講者一覧 | `学员列表` | 全アクティブ受講者 |
| 申込名簿 | `名单 コース名` | コースの申込者一覧 |
| ランダム分組 | `分组 コース名\|组数=N`（`\|每组=M`） | グループ分け結果 |
| 自動リマインド | （自動） | 開講約1時間前に申込者へ Zoom 付き通知 |

- コース名は部分一致。複数ヒット時は具体名を要求。
- 時刻は JST 入力 → UTC 保存 → JST 表示。
- Zoom は `ZOOM_ENABLED` で本番/デモ切替（Server-to-Server OAuth）。

## 3. 人事（jinji）— HR向け

| 機能 | キーワード | 出力 |
|------|-----------|------|
| 提出状況 | `一覧` / `提出状況` | 全員の提出マトリクス（未提出を上に） |
| 未提出確認 | `未提出確認` | 未提出者の抽出（月指定可） |
| 催促 | `催促` / `リマインド` | 未提出者へ LINE push（提出済は除外） |
| 一括DL | `一括DL` | 当月の全提出を ZIP・署名付き短縮リンク |
| 名簿一覧 | `名簿` | 全社員（ID/氏名/部署/役割/紐付け状況） |
| 名簿追加 | `社員追加 氏名 部署 [役割]` | 新規社員を追加 |
| 名簿変更 | `社員変更 番号 項目 値` | 氏名/部署/役割を変更 |
| 名簿削除 | `社員削除 番号` | 削除＋紐付け解除 |
| メール校正 | `メール校正` | 社内校正ツール URL |

- 月指定は末尾に `202606` のように付与（省略時は当月）。
- 名簿変更は **他 HR 全員へ変更通知**を配信。
- 提出ファイルの受領は社員チャネルが担当（人事は集計・管理）。

## 4. 営業（eigyo）— 通勤コスト比較

| 機能 | 入力 | 出力 |
|------|------|------|
| 通勤比較 | `検索路線：駅名`（明示プレフィックス必須） | 待機要員の通勤コスト（定期が安い順）+ 確認URL + 結果CSVボタン |
| 案内 | 駅名のみ等の自由入力 | 操作の案内（誤作動防止のため比較しない） |
| 要員CSV DL | `要員一覧DL` | CSV ダウンロードボタン（全項目引用符） |
| 要員CSV 取込 | CSV ファイル送信 | 反映（楽観ロックで競合検知 / 列ずれ検知 / 住所→駅変換） |
| 単票 追加 | `追加\|番号\|氏名\|最寄駅or住所\|部署\|状態` | upsert |
| 単票 変更 | `変更\|番号\|最寄駅or住所\|状態` | 更新 |
| 単票 削除/照会 | `削除\|番号` / `照会\|番号` | 削除 / 照会 |

- 比較プレフィックス：`検索路線`/`路線検索`/`通勤比較`/`現場`/中文同義語/`route` 等。
- 経由は「駅→[路線名]→駅」、駅間徒歩連絡は〔徒歩〕で明示（乗換回数に含めない）。
- 住所はカンマを含むため CSV では半角 `"` で囲む。`|` 区切りの単票はカンマ安全。
- 住所を入れると **Amazon Location geo-places** で最寄駅へ変換（住所は保存しない）。
- 経路は **DynamoDB キャッシュ**で再取得を抑制。重い取得は ack→push の非同期。

## 6. 非機能要件（抜粋）

- **可用性**：チャネル独立。1 チャネル障害が他へ波及しない。
- **応答性**：重い処理（ekitan 取得・催促送信）は非同期（ack→push）。
- **整合性**：要員 CSV は楽観ロックで同時編集の上書きを防止。
- **セキュリティ**：署名検証、SSM SecureString、占用ロック、役割制御。
- **国際化**：日本語／中文の動的切替、多言語キーワード。

## 7. 運用機能（2026-06 追加）

### 7-1. 登録直後の言語選択
本人確認（登録/紐付け）が成功した直後は、使い方ではなく **言語選択（日本語／中文）** を先に表示し、選択後にメニュー/ヘルプを出す（全チャネル統一）。

### 7-2. 登録解除（多言語）
紐付けを解除するキーワードを多言語対応：日本語（登録解除／紐付け解除／アカウント解除／ログアウト 等）、中文（解绑／解除绑定／取消绑定／登录解除）、英語（unbind／logout／reset）。

### 7-3. 氏名認証の表記ゆれ対応
本人確認の氏名照合は表記ゆれを吸収する：**NFKC（全/半角）＋空白除去＋小文字化＋カタカナ→ひらがな**。さらに花名册の **別名（aliases）** を登録すれば、**簡体字／繁体字／カナ／ハングル／ローマ字** のどの表記でも一致する（例：`孙成功`／`孫成功`／`ソンセイコウ`）。別名は `社員変更 <社員番号> 別名 表記1,表記2,…` で設定。

### 7-4. ブロック/削除の検知
ユーザーが bot をブロック/削除すると LINE は `unfollow` を送る。これを検知して可視化する。

| 機能 | 説明 |
|------|------|
| リアルタイム検知 | 各チャネルが `unfollow` 受信で花名册に `blocked` フラグ＋`[UNFOLLOW]` ログ。`follow`（再追加）で解除 |
| 名簿表示 | 人事「名簿」一覧でブロック中ユーザーを **🚫ブロック中** と表示 |
| 日次リコンサイル | 毎日 **0:00（JST）** に全紐付けを点検。各ユーザーの **役割に対応するチャネル**（社員＝全員／teacher＝研修／hr＝人事／sales・営業部＝営業）の profile API を確認し、対応チャネルすべてで到達不可なら **lineUserId と認証行をクリア** |

> 役割に対応しないチャネルへの到達は「使えない」ため判定に含めない（例：営業部の社員が研修 bot を友だち追加していても、研修は役割ゲートで使えないので活きている証拠にしない）。一時障害・429/5xx は安全側に「到達可能」とみなし誤クリアを防ぐ。

## 8. 提出・催促の拡張（2026-07 追加）

### 8-1. リッチメニュー（全5チャネル・大ボタン化）
`tools/richmenu.py` で PIL 生成 → LINE API で既定メニュー登録。セル全面の大きなカード型ボタン（絵文字不使用・視認性重視）。
| チャネル | ボタン |
|------|------|
| 社員 | 勤怠提出／経費提出／その他経費／履歴／研修／人事 |
| 総務 | 本日認証／未提出確認／催促／催促予約／一覧／一括DL |
| 人事 | 本日認証／未提出確認／一覧／一括DL／メール校正／名簿 |
| 研修 | 本日認証／講師ヘルプ／学員一覧 |
| 営業 | 本日認証／要員リストDL |

### 8-2. PDF・画像での提出（社員）
- 勤怠・経費とも **xlsx / PDF / 画像（JPEG・PNG）** を受付。種別はファイル先頭バイト（マジックナンバー）で判定し、S3 には実形式の拡張子・Content-Type で保存。
- xlsx は従来の内容チェック（年月・氏名・休日勤務・日付欠落・交通費固有）を実施。**PDF/画像はチェック不能のため保存のみ**とし、受付メッセージに「担当者が目視で確認」と明示。
- 「勤怠提出」「経費提出」ボタンで **次のファイルの種別を事前確定**できる（写真などファイル名の無い提出に対応、employees.expectType）。

### 8-3. その他経費（社員）
- 新種別 `other`（宿泊費・接待費など）。「その他経費」→ 用途入力 → ファイル送信（Excel/PDF/画像）の 2 ステップ。
- S3: `hr/{yyyy}/{mm}/others/{用途}_{氏名}_{period}_{ts}.{ext}`。submissions SK=`{period}#other#{ts}` で**月内複数件**可。月次必須提出（勤怠・経費）の未提出判定には含めない。

### 8-4. 手動「済」（総務）
- **未提出確認の一覧に「済 ◯◯さん」ボタン（Quick Reply、先頭13名）**を表示。タップ＝`済 <社員番号>` 送信で即除外（一覧の行頭にも社員番号を表示）。
- コマンドは `済 <氏名|社員番号> [勤怠|経費]`（roster_resolve が氏名/番号どちらも解決）。roster.manualOk に `{period}#{type} → {by, at, note}` を監査情報つきで記録。
- `missing / missing_all_types / roster_status` が manualOk を尊重 → 未提出確認・手動/自動/予約催促のすべてから除外。`一覧` は **✓(手)** 表示。`済解除` で取消。LINE未登録者にも適用可。

### 8-5. 催促予約（総務）
- 「催促予約」→ LINE **日時ピッカー**（postback）→ `bookings` テーブル（`brightstar-soumu-{stage}-bookings`：bookingId, runAtEpoch, runAtJst, status, createdBy）に登録。
- EventBridge **10分間隔**のポーラーが期限到来分を実行（reminder Lambda `trigger:"poll"`）→ 実行時点の未提出者のみに送信 → `status=sent`。`予約確認`（一覧）／`予約取消 <ID>`。
- soumu の line.py に postback 解析・`datetime_picker_message` を追加（他チャネルは従来どおり）。
