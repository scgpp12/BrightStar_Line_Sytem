# 機能仕様書 — BrightStar 統合 LINE アシスタント

版本：v1.0 / 最終更新：2026-06-19

本書は 4 チャネル共通機能と、各チャネル固有機能の仕様をまとめる。

---

## 1. 共通機能

### 1-1. 日次本人確認（認証ゲート）
- 入力「所属部署 お名前」を花名册(roster)と照合し、`role` を判定。
- 役割（role）：`employee` / `hr` / `teacher` / `sales`。チャネルごとに通す役割を限定。
- ゲートの判定結果（action）：

| action | 意味 | 応答 |
|--------|------|------|
| `ok` / `pass` | 認証成功・通過 | 機能利用へ |
| `need_bind` | 未紐付け | 「所属部署 お名前」を要求 |
| `not_found` | 名簿に該当なし | 人事へ連絡を案内 |
| `ambiguous` | 同部署同姓名が複数 | 社員番号の付与を要求 |
| `wrong_role` | 役割が不一致 | 該当チャネルの案内 |
| `tap` | 紐付け済み・当日未認証 | 「認証」タップを促す |
| `taken` | 番号が別アカウントで使用済み | 人事へ連絡を案内 |

- **占用ロック**：`bind_line(empId, userId)` は他の紐付けを解除してから排他的に登録。
- **登録解除**：`unbind(userId)` で花名册の `lineUserId` と認証行をすべてクリア。
- **テスト用バックドア**（人事のみ・任意）：`sonsik<YYYYMMDD>` で 1 時間の臨時 HR 権限（`MASTER_HR_PREFIX` 設定時のみ有効）。

### 1-2. 言語設定
- 認証後その日の最初に「日本語／中文」を選択（既定：日本語）。`auth` テーブルに `lang` / `langDate` を保持。
- キーワードは多言語（日・英・韓・中）の同義語を `resolve()` で canonical intent に解決。

### 1-3. ヘルプ（Quick Reply）
- 機能名とキーワードをボタン化。タップすると当該キーワードが自動送信される。

## 2. 研修（kenshu）— 講師向け

| 機能 | 入力 | 出力 |
|------|------|------|
| コース作成 | `建课 タイトル\|日時\|所要分\|定員\|紹介` | 下書き(draft)作成 |
| コース公開 | `发布 コース名` | Zoom 会議作成 + 公開、主催/受講リンク |
| コース変更 | `改课 コース名\|項目=値;…` | 該当項目を更新 |
| コース削除 | `删课 コース名` | 削除 |
| 受講者一覧 | `学员列表` | 全アクティブ受講者 |
| 申込名簿 | `名单 コース名` | コースの申込者一覧 |
| ランダム分組 | `分组 コース名\|组数=N`（`\|每组=M`） | グループ分け結果 |
| 自動リマインド | （自動） | 開講約1時間前に申込者へ Zoom 付き通知 |

- コース名は部分一致。複数ヒット時は具体名を要求。
- 時刻は JST 入力 → UTC 保存 → JST 表示。
- Zoom は `ZOOM_ENABLED` で本番/デモ切替（Server-to-Server OAuth）。

## 3. 人事（jinji）— HR向け

| 機能 | キーワード | 出力 |
|------|-----------|------|
| 提出状況 | `一覧` / `提出状況` | 全員の提出マトリクス（未提出を上に） |
| 未提出確認 | `未提出確認` | 未提出者の抽出（月指定可） |
| 催促 | `催促` / `リマインド` | 未提出者へ LINE push（提出済は除外） |
| 一括DL | `一括DL` | 当月の全提出を ZIP・署名付き短縮リンク |
| 名簿一覧 | `名簿` | 全社員（ID/氏名/部署/役割/紐付け状況） |
| 名簿追加 | `社員追加 氏名 部署 [役割]` | 新規社員を追加 |
| 名簿変更 | `社員変更 番号 項目 値` | 氏名/部署/役割を変更 |
| 名簿削除 | `社員削除 番号` | 削除＋紐付け解除 |
| メール校正 | `メール校正` | 社内校正ツール URL |

- 月指定は末尾に `202606` のように付与（省略時は当月）。
- 名簿変更は **他 HR 全員へ変更通知**を配信。
- 提出ファイルの受領は社員チャネルが担当（人事は集計・管理）。

## 4. 営業（eigyo）— 通勤コスト比較

| 機能 | 入力 | 出力 |
|------|------|------|
| 通勤比較 | `検索路線：駅名`（明示プレフィックス必須） | 待機要員の通勤コスト（定期が安い順）+ 確認URL + 結果CSVボタン |
| 案内 | 駅名のみ等の自由入力 | 操作の案内（誤作動防止のため比較しない） |
| 要員CSV DL | `要員一覧DL` | CSV ダウンロードボタン（全項目引用符） |
| 要員CSV 取込 | CSV ファイル送信 | 反映（楽観ロックで競合検知 / 列ずれ検知 / 住所→駅変換） |
| 単票 追加 | `追加\|番号\|氏名\|最寄駅or住所\|部署\|状態` | upsert |
| 単票 変更 | `変更\|番号\|最寄駅or住所\|状態` | 更新 |
| 単票 削除/照会 | `削除\|番号` / `照会\|番号` | 削除 / 照会 |

- 比較プレフィックス：`検索路線`/`路線検索`/`通勤比較`/`現場`/中文同義語/`route` 等。
- 経由は「駅→[路線名]→駅」、駅間徒歩連絡は〔徒歩〕で明示（乗換回数に含めない）。
- 住所はカンマを含むため CSV では半角 `"` で囲む。`|` 区切りの単票はカンマ安全。
- 住所を入れると **Amazon Location geo-places** で最寄駅へ変換（住所は保存しない）。
- 経路は **DynamoDB キャッシュ**で再取得を抑制。重い取得は ack→push の非同期。

## 5. 社員（shain）— 一般社員向け統合フロント

| 機能 | 入力 | 出力 |
|------|------|------|
| 本人登録 | `所属部署 お名前` | 在籍者として登録 |
| 日次モード選択 | （自動） | 「研修／人事」チューザー（毎日） |
| 研修モード | `研修` | 受講・申込（kenshu へ委譲、受講者を自動プロビジョニング） |
| 人事モード | `人事` / Excel 送信 | 提出・テンプレ・履歴（jinji へ委譲） |
| ヘルプ | `ヘルプ` | 機能ボタン |
| 登録解除 | `登録解除` | 紐付け解除 |

## 6. 非機能要件（抜粋）

- **可用性**：チャネル独立。1 チャネル障害が他へ波及しない。
- **応答性**：重い処理（ekitan 取得・催促送信）は非同期（ack→push）。
- **整合性**：要員 CSV は楽観ロックで同時編集の上書きを防止。
- **セキュリティ**：署名検証、SSM SecureString、占用ロック、役割制御。
- **国際化**：日本語／中文の動的切替、多言語キーワード。
