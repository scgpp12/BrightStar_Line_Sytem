# 人事アシスタント（HR向け）利用マニュアル

**BrightStar 統合 LINE アシスタント — 人事チャネル**
対象：人事（HR）担当者 / 版本：v1.0 / 最終更新：2026-06-19

> 本チャネルは **人事担当（HR）専用** です。社員からの勤怠・通勤費の提出は「BrightStar 社員アシスタント」が受け付けます。
> 中文：本窗口仅限**人事(HR)**使用；员工提交考勤·通勤费请用「社員アシスタント」。

---

## 1. はじめに（毎日の本人確認）

人事情報を扱うため、**毎日（東京時間）最初の利用時に本人確認**が必要です。

### 1-1. 初回登録（最初の1回だけ）
```
人事部 鈴木
```
- 社員名簿と照合し、**役割が HR（人事）** の本人のみ登録できます。
- 同姓同名は社員番号を付与：`人事部 鈴木 E001`
- 一度登録すると LINE アカウントと社員番号が**専用に紐付き**ます。

### 1-2. 毎日の認証
2日目以降は **「認証」** ワンタップ（またはメニューの「本日認証」）でその日の確認が完了します。

### 1-3. 言語の選択
認証後、最初に **「日本語」／「中文」**（既定：日本語）。後から「日本語」「中文」で切替可。

### 1-4. 登録のやり直し
**「登録解除」** で紐付けを解除し、別人で登録し直せます。

## 2. コマンド一覧

| 操作 | 主なキーワード | 内容 |
|------|---------------|------|
| ヘルプ | `ヘルプ` / `菜单` | 機能とキーワードをボタン表示 |
| 提出状況の確認 | `一覧` / `提出状況` / `提交情况` | 全社員の提出状況（未提出を上に） |
| 未提出者の確認 | `未提出確認` / `未提出` / `未提交` | 未提出者だけを抽出 |
| 催促（リマインド） | `催促` / `リマインド` / `催办` | 未提出者へ LINE 通知を送信 |
| 一括ダウンロード | `一括DL` / `打包下载` | 当月の全提出を ZIP でDL |
| 名簿管理 | `名簿` / `花名册` ほか | 社員名簿の閲覧・追加・変更・削除 |
| メール校正ツール | `メール校正` / `邮件校正` | 社内メール自動校正ツールのリンク |
| 本日認証 | `認証` / `本日認証` | その日の本人確認 |
| 登録解除 | `登録解除` | 別人で認証し直す |

> 多くのコマンドは **月指定**が可能です（例：`未提出確認 202606`、`一括DL 202606`）。省略時は当月。

---

## 3. 操作の詳細

### 3-1. 提出状況の確認
- `一覧` … 全員の提出マトリクス（✓＝提出済 / ✗＝未提出）。件数が多い場合は分割して送信します。
- `未提出確認` … 未提出者の氏名・部署・未提出の種別を一覧。

### 3-2. 催促（リマインド）
```
催促            ← 当月の未提出者へ
催促 202606     ← 月を指定
```
- 未提出者にだけ LINE 通知が届きます（提出済の人は自動で除外）。
- 通知文例：`【リマインド】{月} の {種別} が未提出です。期日までにご提出ください。`

### 3-3. 一括ダウンロード
```
一括DL
```
- 当月の全提出ファイルを **ZIP** にまとめ、「全件ダウンロード」ボタンを返します。
- リンクは署名付きの短縮 URL で、一定時間で失効します。

### 3-4. 社員名簿の管理（CRUD）
| 操作 | 送信例 | 内容 |
|------|--------|------|
| 一覧 | `名簿` | 全社員（ID／氏名／部署／役割／LINE紐付け状況） |
| 追加 | `社員追加 山田太郎 開発部 hr` | 新規社員を追加（役割省略時は一般社員） |
| 変更 | `社員変更 E003 部署 営業部` | 氏名／部署／役割／別名を変更 |
| 別名設定 | `社員変更 E001 別名 孫成功,ソンセイコウ` | 認証で使える氏名の別表記（簡体/繁体/カナ/ハングル/ローマ字） |
| 削除 | `社員削除 E003` | 名簿から削除＋LINE紐付け解除 |

> **氏名の表記ゆれ対応**：認証の「所属部署 お名前」は、全角/半角・かな（ソン＝そん）・大小文字を自動で同一視します。さらに **別名（aliases）** を登録すれば、簡体字「孙成功」／繁体字「孫成功」／カナ「ソンセイコウ」など複数表記のどれでも本人確認できます。中国籍の社員などは別名にフリガナ・繁体字を入れておくと便利です。

- 役割（role）の値：`employee`（一般）／`hr`（人事）／`teacher`（講師）／`sales`（営業）。
- 名簿を変更すると、**他の HR 全員に変更通知**が自動配信されます（変更者と内容入り）。
- 一覧に **🚫ブロック中** が付く人は、その LINE アカウントが bot をブロック/削除しています。毎日 0:00 に自動点検し、使えるアシスタント全てで到達不可なら紐付け（LINE登録）が自動で解除されます（本人が再追加・再登録すれば元通り）。

### 3-5. メール校正ツール
```
メール校正
```
社内向けの **メール自動校正ツール**（社内ローカル AI）の URL を返します。ブラウザで開いて利用してください。

---

## 4. 提出テンプレートの配布について

社員が提出に使う **勤怠・通勤費のテンプレート** は社員アシスタント側から配布されます。人事チャネルは提出の**受領は行わず**、状況確認・集計・催促・名簿管理に専念します（責務分離）。

---

## 5. よくある質問

**Q. 社員が送った Excel をこの窓口に転送できますか。**
できません。提出は社員アシスタントが受領します。人事チャネルは管理・集計用です。

**Q. 催促を送ったのに届かない人がいます。**
LINE の友だち追加・通知設定をご確認ください。提出済の人にはそもそも送られません。

**Q. 名簿の役割を間違えました。**
`社員変更 社員番号 役割 hr` のように修正できます。
